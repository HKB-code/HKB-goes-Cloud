🎯 Why Bitwise Operators Are Used
Bitwise operators are used because they let you manipulate data at the binary level — which is:
- Fast: Direct CPU instructions, faster than arithmetic or higher-level logic.
- Compact: Store multiple flags/permissions in a single integer.
- Precise: Control individual bits for low-level tasks.

Common Uses
- Flags & Permissions:
Store multiple boolean values in one integer.
Example: Read=0001, Write=0010, Execute=0100.
Combine them with |, check with &, remove with &^.
- Masks:
Extract or clear specific bits.
Example: value & 0xFF → keeps only the last 8 bits.
- Toggling:
Flip certain bits using XOR (^).
Example: flags ^ Write → toggles the Write permission.


🔀 Why Bit Shifting Is Used
Shifting is used because it’s equivalent to multiplying or dividing by powers of 2, but much faster at the CPU level.
Common Uses
- Fast Math:
x << 1 = x * 2, x >> 1 = x / 2.
Used in performance-critical code.
- ID Generation:
Systems like Snowflake IDs use shifting to pack timestamp, machine ID, and sequence into one integer.
- Encoding/Decoding:
Shifts help place values into specific bit positions.
Example: ip := (192 << 24) | (168 << 16) | (1 << 8) | 5 → packs an IP address into a 32-bit integer.
- Compression:
Efficiently store multiple small values inside one integer.


🖥️⚙️ Practical Example in Go: Permissions:

const (
    Read  = 1 << iota // 0001
    Write             // 0010
    Execute           // 0100
)

func main() {
    userPerm := Read | Write // 0011 (user can read & write)

    // Check permission
    if userPerm & Write != 0 {
        fmt.Println("User can write")
    }

    // Remove Write
    userPerm = userPerm &^ Write // 0001
    fmt.Println("Updated permissions:", userPerm)
}

🚀 Summary
- Bitwise ops → control individual bits (flags, masks, toggles).
- Bit shifts → fast multiply/divide by powers of 2, pack/unpack data.
- They’re used in backend systems, OS kernels, databases, networking, cryptography, and scalable ID generation.


# Notes:
🔎 What is iota?
- iota is a special identifier in Go used inside const declarations.
- It represents successive integer constants, starting from 0 and increasing by 1 for each line in the same const block.
- Think of it as a counter that resets to 0 whenever a new const block begins.

🚀 Why Use iota?
- Cleaner code: No need to manually assign numbers.
- Less error-prone: Auto-increment avoids mistakes in numbering.
- Flexible: Works with bit shifting, arithmetic, and skipping.
- Common use cases:
- Enumerations (days of week, states, modes).
- Flags/permissions (bitmasking).
- Scaling constants (KB, MB, GB).

🚀 Practical Use
This pattern is often used to define data size constants
const (
    KB = 1 << (10 * iota) // 1024
    MB                    // 1048576
    GB                    // 1073741824
    TB                    // 1099511627776
)
